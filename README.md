# 🤖 llm-learning-rag — 基于 RAG 的大模型知识问答助手

> 把零散的大模型学习资料（教程章节 + 经典论文）整理成可检索的知识库，回答时**带出处、不编造**——用检索增强生成（RAG）从源头抑制大模型幻觉。

---

## 📌 项目简介

大语言模型回答专业问题时容易"一本正经地胡说"（幻觉），根因是它在知识空白处凭统计模式硬编。本项目用 **RAG（检索增强生成）** 解决：先把可信资料检索出来、注入提示词，让模型"看着资料答"，并标注答案来源，使回答**可核查**。

知识库聚焦**大模型 / AI 领域**（LLM 教程、经典论文），因此它既是一个工程作品，也是作者自己的"AI 知识外脑"——测试它的每个问题，同时也在巩固大模型知识。

## ✨ 功能特性

- 📂 **文档摄入与切分**：加载多份资料，切成合适的文本块
- 🔎 **语义向量检索**：基于 embedding 的检索，Chroma 持久化向量库（建一次、永久复用）
- 💬 **带来源的问答**：回答附带检索到的原文出处，可追溯、可信任
- 🚧 **（规划）重排序 rerank**：两阶段检索（粗召回 + 精排），提升相关性
- 🚧 **（规划）检索质量评估**：用量化指标衡量回答好坏
- 🚧 **（规划）Web 交互界面**

## 🛠 技术栈

| 类别 | 选型 |
| --- | --- |
| 语言 | Python 3.10+ |
| 大模型 | 智谱 GLM（`glm-4.7-flash` 生成 / `embedding-3` 向量，1024 维） |
| RAG 框架 | LangChain |
| 向量库 | Chroma（本地持久化） |
| 规划中 | rerank 重排序、检索评估、Streamlit / Gradio UI |

## 💡 实现亮点 / 设计取舍



- **从零手写 → 框架重构**：检索核心（embedding 编码、余弦相似度、Top-K 选取）先用 numpy [**从零手写实现**](./rag_scratch.py) 以吃透原理，再用 LangChain + Chroma [**重构为工程版**](./rag.py)。框架对本项目不是黑箱——`similarity_search()` 底下就是手写版里的"算余弦 + argsort 取 topk"，每一行调用底下发生了什么都清楚。
- **为什么选 RAG 而非微调**：目标是让模型用上"最新且可信的外部资料"。RAG 无需训练、资料可随时更新、答案可溯源，比微调更轻、更适合知识问答场景。
- **为什么标注来源**：可追溯 = 可信任，也便于定位"模型是基于哪段资料回答的"，从而发现检索环节的问题。

## 📁 项目结构

```
llm-learning-rag/
├── data/              # 知识库原始语料（本地）
├── chroma_db/         # 向量库（自动生成，不入库）
├── rag_scratch.py     # 从零手写版（numpy 实现，展示原理）
├── rag.py             # 框架版（LangChain + Chroma）
├── requirements.txt   # 依赖
├── .env               # API Key（不入库）
├── .gitignore
└── README.md
```

## 🚀 快速开始

```bash
# 1. 克隆
git clone <仓库地址>
cd llm-learning-rag

# 2. 装依赖
pip install -r requirements.txt

# 3. 配置 API Key：在项目根目录新建 .env，写入
#    ZHIPUAI_API_KEY=你的智谱key

# 4. 运行
python rag.py
```
